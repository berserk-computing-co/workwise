# Phase 2 — PDF Estimate Generation

**Size:** M
**Depends on:** Phase 1 (logo URL, phone, license number on Org)
**Unlocks:** Phase 3 (email attaches the PDF buffer from `PdfService.renderToBuffer()`)

## Why Before Email

Phase 3 attaches the PDF to the bid email. `PdfService.renderToBuffer()` is a shared utility called by both the download endpoint and the email sender — building the PDF module first means Phase 3 gets it for free, with no second DB round-trip.

---

## Approach

Server-side NestJS rendering using `@react-pdf/renderer` (already in `package.json`, currently unused). Returns a `Buffer`, streamed as `application/pdf`. Fetch org logo as a **base64 data URI** to avoid `canvas`/DOM dependencies in Node.

---

## New API Module: `apps/api/src/pdf/`

```
apps/api/src/pdf/
  pdf.module.ts
  pdf.service.ts
  pdf.controller.ts
  templates/
    estimate.template.tsx
```

### `pdf.service.ts`

```typescript
renderToBuffer(project: Project, organization: Organization): Promise<Buffer>
```

1. If `organization.logoUrl`, fetch via `axios` and convert to base64 data URI string
2. Call `renderToStream()` from `@react-pdf/renderer` with `<EstimateTemplate>` component
3. Pipe stream to `Buffer`, return it

Called directly by `pdf.controller.ts` for downloads and by `email.service.ts` (Phase 3) for attachments.

### `pdf.controller.ts`

```
GET /projects/:id/pdf
```

- Load project + org (with sections → items → options)
- Call `pdfService.renderToBuffer()`
- Set headers:
  - `Content-Type: application/pdf`
  - `Content-Disposition: attachment; filename="estimate-<projectId>.pdf"`
- Stream buffer as response

### `templates/estimate.template.tsx`

`@react-pdf/renderer` JSX component. Sections:

| Section | Content |
|---------|---------|
| **Header** | Contractor logo (or name if no logo), license number, phone, website |
| **Client block** | Client name (if set), project address, date generated |
| **Sections table** | Description \| Qty \| Unit \| Unit Cost \| Extended Cost — one row per `EstimateItem` |
| **Options block** | Good / Better / Best total rows |
| **Footer** | "Generated by [Org Name]" + page numbers |

Styles: neutral palette, clean sans-serif (PDF-safe font — Helvetica). No WorkWise branding.

## Frontend Changes

### Project detail page (`/projects/[id]/page.tsx`)

Add a **"Download PDF"** button in the project header actions row (alongside the existing Generate / Cancel buttons).

```typescript
async function handleDownloadPdf() {
  const res = await fetch(`/api/proxy/projects/${projectId}/pdf`);
  const blob = await res.blob();
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = `estimate-${projectId}.pdf`;
  a.click();
  URL.revokeObjectURL(url);
}
```

Button state:
- Disabled if project has no sections/items yet (no content to render)
- Shows a spinner while fetching (PDF generation may take 1–3s)
